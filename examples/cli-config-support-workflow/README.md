# CLI Config Support Workflow Example

This example shows how to use all four skills for a backend or tooling-oriented feature:

- add configuration file support to an existing CLI tool
- preserve existing command behavior
- define precedence rules clearly
- avoid implementation drift while changing a multi-module toolchain

Workflow:

1. requirements
2. design
3. tasks
4. implementation

Files in this directory:

- `01-requirements.md`
- `02-design.md`
- `03-tasks.md`
- `04-implementation.md`
