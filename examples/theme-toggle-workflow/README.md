# Theme Toggle Workflow Example

This example shows how to use all four skills in order with one realistic feature:

- add light/dark theme switching
- remember the user's choice
- avoid implementation drift

Workflow:

1. requirements
2. design
3. tasks
4. implementation

Files in this directory:

- `01-requirements.md`
- `02-design.md`
- `03-tasks.md`
- `04-implementation.md`
