# Legacy Module Refactor Workflow Example

This example shows how to use all four skills for a maintenance-oriented refactor:

- refactor an existing oversized module
- split responsibilities into service + adapter + policy
- preserve existing behavior while improving structure
- avoid drifting into accidental redesign during coding

Workflow:

1. requirements
2. design
3. tasks
4. implementation

Files in this directory:

- `01-requirements.md`
- `02-design.md`
- `03-tasks.md`
- `04-implementation.md`
